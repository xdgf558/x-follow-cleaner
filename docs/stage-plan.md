# Stage Plan

本项目必须按阶段推进。每个阶段只实现该阶段明确列出的功能，发现新需求时先记录到 `docs/known-issues.md` 或本文件，不直接扩展。

## 安全边界

1. 不自动取关。
2. 不自动关注。
3. 不读取 Cookie、Token、密码、私信。
4. 不上传任何数据。
5. 不绕过 X 限制。
6. 不高速自动访问页面。
7. 不默认开启批量检查。
8. 不加入未经说明的新权限。

## v1.1 低频批量检查边界

1. 默认关闭。
2. 用户必须在设置页主动开启。
3. 每批最多 20 个账户。
4. 每个账户间隔至少 15 秒。
5. 每天最多检查 100 个账户。
6. 运行时显示当前正在检查的账户。
7. 用户可以暂停。
8. 出现验证或访问限制必须立即停止。
9. 不得自动取关。
10. 不得后台静默读取；运行时必须打开可见的 X 主页。
