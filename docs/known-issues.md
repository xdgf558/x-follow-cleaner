# Known Issues

## 当前已知问题

1. 真实 X 页面 DOM 结构可能变化，扫描和解析逻辑需要在真实页面中人工验证。
2. 中文或其他语言的 X 时间格式不作为 MVP 优先支持范围；如果 `time` 标签缺少 `datetime`，建议使用英文界面检测最近发帖时间。
3. 真实 Chrome Load unpacked、Popup Console 和按钮交互仍需人工安装后验证。
4. Following 页面扫描选择器基于当前可见 DOM 和常见 `data-testid`，需要真实 X 页面回归验证。
5. 主页活跃度读取已限制为当前账号自己的 `/用户名/status/...` 时间链接，但 X DOM 变化仍可能影响准确性。
6. X 的验证、访问限制和错误页面文案可能变化；插件会保守停止并提示，但仍需要真实页面回归。
7. 低频自动检查已改为后台队列和 `chrome.alarms` 调度；结果页切到后台时仍会继续，但 Chrome 退出、电脑休眠、扩展重载或检查标签页关闭仍可能导致延后或停止。
8. 低频自动检查会逐个打开可见 X 主页，仍建议小批量使用并观察账号状态。
9. 中英文切换覆盖插件主要界面；X 页面自身返回的异常文案仍可能来自 X 页面语言或 DOM 内容。

## 禁止直接加入 MVP 的功能

1. 自动取关。
2. 自动关注。
3. 自动高速或静默批量打开账户主页。
4. 自动高速滚动 Following 页面。
5. 自动绕过 X 限制。
6. 云端同步。
7. 登录系统。
8. X 官方 API 模式。
