# X Follow Cleaner

X Follow Cleaner 是一个本地运行的 Chrome 插件，用于辅助你检查 X 关注列表中疑似长期未活跃的账户。

插件只读取你当前浏览器页面中已经展示出来的公开内容，并把扫描结果保存在本机 `chrome.storage.local`。它不会自动取关、不会自动关注、不会读取 Cookie、Token、密码或私信，也不会上传你的关注列表。

## 快速安装

普通用户推荐安装打包版：

1. 打开 [GitHub Releases](https://github.com/xdgf558/x-follow-cleaner/releases)。
2. 下载最新的 `x-follow-cleaner-vX.Y.Z.zip`。
3. 解压 zip。
4. 打开 Chrome，进入 `chrome://extensions`。
5. 打开右上角 Developer Mode。
6. 点击 Load unpacked。
7. 选择解压后的插件文件夹。
8. 打开 `x.com` 登录后开始使用。

## 核心特点

1. 本地运行。
2. 数据保存在 `chrome.storage.local`。
3. 不上传关注列表。
4. 不读取 Cookie、Token、密码、私信。
5. 不自动取关。
6. 不自动关注。
7. 不批量点击。
8. 不自动滚动 X 页面。
9. 不使用 X 官方 API。
10. 支持中文和 English 界面切换。

## 使用方法

1. 打开 `x.com` 并登录。
2. 进入你的 Following 页面，格式通常是 `https://x.com/{你的用户名}/following`。
3. 手动向下滚动页面，让更多关注账户加载出来。
4. 点击 Chrome 工具栏中的插件图标。
5. 点击“扫描当前页面”。
6. 点击“查看结果”。
7. 在结果页点击某个账户的“打开主页”。
8. 在该账户主页中再次点击插件图标。
9. 点击“读取当前主页最近发帖时间”。
10. 回到结果页查看状态。
11. 手动决定是否在 X 页面取关。
12. 对不想重复处理的账户，可以标记“已处理”或加入“白名单”。

## 使用建议

1. 每次只扫描当前已经展示出来的账户。
2. 不要长时间连续高频操作。
3. 遇到验证或访问限制时立即停止。
4. 取关前请手动打开主页确认。
5. 白名单用于保留重要账户。
6. 已处理用于避免重复检查。
7. 建议将 X 页面语言切换为英文后读取最近发帖时间。

## 设置

点击 Popup 中的“打开设置”可以调整：

1. 插件界面语言：中文 / English。
2. 未活跃判断天数：30 / 60 / 90 / 180。
3. 是否隐藏白名单账户。
4. 是否显示无法判断账户。
5. 默认排序。
6. X 页面语言提示。
7. 保守验证节奏。

设置只保存在本地，不会上传。

## 保守验证节奏

设置页默认开启“保守验证节奏”。开启后，两次“扫描当前页面”之间必须等待至少 30 秒。

你也可以在设置页把等待时间改为 45 秒或 60 秒。Popup 会在按钮上显示剩余秒数，例如“扫描当前页面（18s）”。

建议保持开启，尤其是在首次验证插件或遇到 X 页面加载不稳定时。

## 低频自动检查

低频自动检查默认关闭。开启后，结果页会出现“开始低频检查”按钮。

使用方式：

1. 先在 Following 页面手动滚动并扫描账户。
2. 打开结果页。
3. 打开设置页，开启“低频自动检查”。
4. 回到结果页，点击“开始低频检查”。
5. 插件会从未处理、非白名单、尚未判断的账户中选择一批。
6. 插件会逐个打开账户主页并读取当前可见的最近公开发帖时间。
7. 你可以停留在 X 主页观察检查过程，也可以切回结果页查看进度。
8. 你可以随时回到结果页点击“暂停”。

从 `v0.4.0` 起，低频自动检查的队列由后台 service worker 和 `chrome.alarms` 调度，不再依赖结果页自己的计时器。也就是说，你停留在正在检查的 X 页面时，队列仍会按低频间隔继续推进。

安全限制：

1. 默认关闭，必须手动开启。
2. 每批最多 20 个账户。
3. 每个账户之间至少等待 15 秒。
4. 每天最多检查 100 个账户。
5. 运行时会打开可见的 X 主页，不会后台静默运行。
6. 出现验证或访问限制会立即停止。
7. 不会自动取关、关注或点击 X 操作按钮。
8. Chrome 退出、电脑休眠、扩展被重载或检查标签页被关闭时，当前批次可能暂停、延后或停止。

准确性说明：

1. 插件只读取当前主页中属于该账号自己的 `/用户名/status/...` 发帖时间。
2. 插件会跳过置顶帖，避免把长期置顶的旧内容当成最近活跃时间。
3. 自动检查会等待目标主页切换稳定后再保存结果，减少读到上一个账号页面残留内容的概率。
4. 如果当前可见内容里没有稳定找到该账号自己的公开帖子时间，会标记为无法判断，而不是强行保存旧时间。
5. 第一次读到超过阈值的账号会标记为“待复核未活跃”；第二次重新检查仍然超过阈值后，才标记为“已确认未活跃”。
6. 插件会同时读取主页头部是否显示“跟随你 / Follows you”，用于判断对方当前是否关注你。
7. 如果历史上读到“关注你”，之后再次读取变成“未关注你”，结果页会标记为“疑似取关”。
8. 结果页会显示读取依据，包括原始时间文本、依据帖子链接和检查时间。

重新检查：

1. 每个账户行都有“重新检查”按钮，可单独复核。
2. 开启低频自动检查后，结果页可批量“复核未活跃”和“重查无法判断”。
3. 批量复核仍使用低频间隔和每日上限，不会自动取关或关注。

## 互关状态和疑似取关

插件会在读取账户主页时检查主页头部是否出现“跟随你 / Follows you”一类标记，并在结果页显示：

1. 关注你：当前主页读到对方关注你。
2. 未关注你：当前主页未读到对方关注你。
3. 疑似取关你：之前读到对方关注你，这次读到未关注你。
4. 未检查：还没有读取过该账户主页，或当前页面无法稳定判断。

这个功能只做本地辅助判断。X 页面语言、页面加载、账号保护、异常页面或 DOM 变化都可能影响结果；“疑似取关”建议再手动打开主页确认一次。

从 `v0.5.1` 起，如果主页头部已经加载但帖子时间线还没加载完成，插件会继续等待最近发帖时间，不会只保存互关状态就结束本次检查。

从 `v0.6.0` 起，插件增加了更多 DOM fallback 和时间格式支持。如果 X 改掉常见 `data-testid`，插件会尝试从可见主页链接和状态帖时间链接中兜底读取；时间解析也补充了周、月、年、`Mar 2025`、`5月30日`、`2025年5月30日` 等格式。

## 数据导出

结果页支持导出 CSV 和 JSON。

CSV 字段：

```csv
username,displayName,profileUrl,lastPostAt,inactiveDays,inactiveConfirmationCount,status,mutualFollowStatus,followsYouLastCheckedAt,followsYouSourceText,suspectedUnfollow,suspectedUnfollowAt,lastSourceText,lastStatusUrl,processed,whitelisted,collectedAt,lastCheckedAt,manualNote
```

JSON 包含本地账户数组、设置和导出时间，不包含 Cookie、Token、密码、私信或浏览器会话信息。

Popup 中点击“清空本地数据”可以清空本地保存的账户列表。此操作只影响插件本地数据，不会影响你的 X 账号。

## 常见问题

### 插件会自动取关吗？

不会。插件只做辅助整理，所有取关都必须由你在 X 页面手动决定和操作。

### 插件会读取我的 Cookie、Token、密码或私信吗？

不会。Manifest 没有申请 `cookies`、`webRequest`、`debugger`、`history`、`downloads` 等高风险权限。

### 为什么插件需要 alarms 权限？

`alarms` 只用于低频自动检查的账户间隔调度，避免结果页切到后台后计时器被 Chrome 降速导致检查卡住。它不会上传数据，也不会绕过 X 的访问限制。

### 停留在 X 页面或最小化 Chrome 时会继续检查吗？

停留在 X 页面时会继续，因为队列在后台调度。Chrome 最小化时通常也会继续，但速度可能受系统节能、网络和 Chrome 后台策略影响；电脑休眠或 Chrome 退出时不能保证继续。

### 为什么扫描不到账户？

可能原因：

1. 当前页面不是 Following 页面。
2. Following 页面还没有加载账户。
3. 你还没有登录 X。
4. X 出现验证或访问限制。
5. X 页面 DOM 结构变化。

### 为什么主页最近发帖时间无法判断？

可能原因：

1. 账户没有公开帖子。
2. 账户受保护。
3. 账户被暂停或不存在。
4. 当前可见内容只有转发、引用或置顶内容，没有找到属于该账号自己的公开帖子时间。
5. 页面语言不是英文，时间格式暂未覆盖。
6. X 页面结构变化。

### 为什么“未关注你”不一定等于“取关你”？

因为插件只能读取当前页面是否显示“跟随你 / Follows you”。如果以前没有记录到“关注你”，那第一次读到“未关注你”只能说明当前未互关；只有历史上读到过“关注你”，之后再读到“未关注你”，才会标记为“疑似取关”。

### 可以批量自动检查所有账户吗？

当前版本支持“低频自动检查”，但它默认关闭，并且不是高速批量工具。开启后插件会逐个打开可见 X 主页读取公开发帖时间，每批最多 20 个账户，每天最多 100 个账户，遇到验证或访问限制会停止。

### 个人网站应该放哪种下载方式？

推荐个人网站放 GitHub Releases 最新版下载链接，同时保留 GitHub 仓库链接。这样用户下载的是干净打包版，你也不用手动维护多个下载文件。
