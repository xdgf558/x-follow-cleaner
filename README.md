# X Follow Cleaner

X Follow Cleaner 是一个本地运行的 Chrome 插件，用于辅助你检查 X 关注列表中疑似长期未活跃的账户。

插件只读取你当前浏览器页面中已经展示出来的公开内容，并把扫描结果保存在本机 `chrome.storage.local`。它不会自动取关、不会自动关注、不会读取 Cookie、Token、密码或私信，也不会上传你的关注列表。

## 核心特点

1. 本地运行。
2. 数据保存在 `chrome.storage.local`。
3. 不上传关注列表。
4. 不读取 Cookie、Token、密码、私信。
5. 不自动取关。
6. 不自动关注。
7. 不批量点击。
8. 不自动滚动 X 页面。
9. 不使用 X 官方 API。

## 下载打包版安装方法

推荐普通用户使用 GitHub Releases 打包版：

1. 打开 [GitHub Releases](https://github.com/xdgf558/x-follow-cleaner/releases)。
2. 下载最新版本里的 `x-follow-cleaner-vX.Y.Z.zip`。
3. 解压 zip 文件。
4. 打开 Chrome。
5. 地址栏输入 `chrome://extensions`。
6. 打开右上角 Developer Mode。
7. 点击 Load unpacked。
8. 选择解压后的插件文件夹，不要直接选择 zip 文件。
9. 插件安装完成。

## 使用方法

1. 打开 `x.com` 并登录。
2. 进入你的 Following 页面，格式通常是 `https://x.com/{你的用户名}/following`。
3. 手动向下滚动页面，让更多关注账户加载出来。
4. 点击 Chrome 工具栏中的插件图标。
5. 点击“扫描当前页面”。
6. 点击“查看结果”。
7. 在结果页点击某个账户的“打开主页”。
8. 在该账户主页中再次点击插件图标。
9. 点击“读取当前主页最近发帖时间”。
10. 回到结果页查看状态。
11. 手动决定是否在 X 页面取关。
12. 对不想重复处理的账户，可以标记“已处理”或加入“白名单”。

## 使用建议

1. 每次只扫描当前已经展示出来的账户。
2. 不要长时间连续高频操作。
3. 遇到验证或访问限制时立即停止。
4. 取关前请手动打开主页确认。
5. 白名单用于保留重要账户。
6. 已处理用于避免重复检查。
7. 建议将 X 页面语言切换为英文后读取最近发帖时间。

## 设置

点击 Popup 中的“打开设置”可以调整：

1. 未活跃判断天数：30 / 60 / 90 / 180。
2. 是否隐藏白名单账户。
3. 是否显示无法判断账户。
4. 默认排序。
5. X 页面语言提示。
6. 保守验证节奏。

设置只保存在本地，不会上传。

## 保守验证节奏

设置页默认开启“保守验证节奏”。开启后，两次“扫描当前页面”之间必须等待至少 30 秒。

你也可以在设置页把等待时间改为 45 秒或 60 秒。Popup 会在按钮上显示剩余秒数，例如“扫描当前页面（18s）”。

建议保持开启，尤其是在首次验证插件或遇到 X 页面加载不稳定时。

## 低频自动检查

低频自动检查默认关闭。开启后，结果页会出现“开始低频检查”按钮。

使用方式：

1. 先在 Following 页面手动滚动并扫描账户。
2. 打开结果页。
3. 打开设置页，开启“低频自动检查”。
4. 回到结果页，点击“开始低频检查”。
5. 插件会从未处理、非白名单、尚未判断的账户中选择一批。
6. 插件会逐个打开账户主页并读取当前可见的最近公开发帖时间。
7. 你可以随时回到结果页点击“暂停”。

安全限制：

1. 默认关闭，必须手动开启。
2. 每批最多 20 个账户。
3. 每个账户之间至少等待 15 秒。
4. 每天最多检查 100 个账户。
5. 运行时会打开可见的 X 主页，不会后台静默运行。
6. 出现验证或访问限制会立即停止。
7. 不会自动取关、关注或点击 X 操作按钮。

## 数据导出

结果页支持导出 CSV 和 JSON。

CSV 字段：

```csv
username,displayName,profileUrl,lastPostAt,inactiveDays,status,processed,whitelisted,collectedAt,lastCheckedAt,manualNote
```

JSON 包含本地账户数组、设置和导出时间，不包含 Cookie、Token、密码、私信或浏览器会话信息。

## 清空数据

Popup 中点击“清空本地数据”可以清空本地保存的账户列表。此操作只影响插件本地数据，不会影响你的 X 账号。

## 常见问题

### 插件会自动取关吗？

不会。插件只做辅助整理，所有取关都必须由你在 X 页面手动决定和操作。

### 插件会读取我的 Cookie、Token、密码或私信吗？

不会。Manifest 没有申请 `cookies`、`webRequest`、`debugger`、`history`、`downloads` 等高风险权限。

### 为什么扫描不到账户？

可能原因：

1. 当前页面不是 Following 页面。
2. Following 页面还没有加载账户。
3. 你还没有登录 X。
4. X 出现验证或访问限制。
5. X 页面 DOM 结构变化。

### 为什么主页最近发帖时间无法判断？

可能原因：

1. 账户没有公开帖子。
2. 账户受保护。
3. 账户被暂停或不存在。
4. 页面语言不是英文，时间格式暂未覆盖。
5. X 页面结构变化。

### 可以批量自动检查所有账户吗？

当前版本支持“低频自动检查”，但它默认关闭，并且不是高速批量工具。开启后插件会逐个打开可见 X 主页读取公开发帖时间，每批最多 20 个账户，每天最多 100 个账户，遇到验证或访问限制会停止。

### 个人网站应该放哪种下载方式？

推荐个人网站放 GitHub Releases 最新版下载链接，同时保留 GitHub 仓库链接。这样用户下载的是干净打包版，你也不用手动维护多个下载文件。

## 账号风险提醒

本项目不能承诺完全没有账号风险。为了降低风险，请保持低频、手动、只读、可见操作。出现验证、访问限制或页面异常时，请立即停止使用插件。

## 开发检查

本项目不需要构建步骤。可以运行：

```sh
npm run check
```

该命令会检查必需文件、Manifest 禁止权限和 `src` 下 JavaScript 语法。

生成 GitHub Releases 或个人网站可用的 zip 打包文件：

```sh
npm run package
```

打包文件会生成到 `dist/x-follow-cleaner-vX.Y.Z.zip`，zip 根目录包含 `manifest.json`，可用于 Chrome 开发者模式安装，也可作为 Chrome Web Store 上传前的基础包。
