# X Follow Cleaner

X Follow Cleaner 是一个本地运行的 Chrome 插件，用于辅助你检查 X 关注列表中疑似长期未活跃的账户。

插件只读取你当前浏览器页面中已经展示出来的公开内容，并把扫描结果保存在本机 `chrome.storage.local`。它不会自动取关、不会自动关注、不会读取 Cookie、Token、密码或私信，也不会上传你的关注列表。

## 核心特点

1. 本地运行。
2. 数据保存在 `chrome.storage.local`。
3. 不上传关注列表。
4. 不读取 Cookie、Token、密码、私信。
5. 不自动取关。
6. 不自动关注。
7. 不批量点击。
8. 不自动滚动 X 页面。
9. 不使用 X 官方 API。

## 本地安装方法

1. 下载项目文件夹。
2. 打开 Chrome。
3. 地址栏输入 `chrome://extensions`。
4. 打开右上角 Developer Mode。
5. 点击 Load unpacked。
6. 选择本项目文件夹。
7. 插件安装完成。
8. 打开 `x.com` 并登录自己的账号。
9. 进入自己的 Following 页面。
10. 手动滚动页面。
11. 点击插件图标。
12. 点击“扫描当前页面”。

## 使用方法

1. 打开 `x.com` 并登录。
2. 进入你的 Following 页面，格式通常是 `https://x.com/{你的用户名}/following`。
3. 手动向下滚动页面，让更多关注账户加载出来。
4. 点击 Chrome 工具栏中的插件图标。
5. 点击“扫描当前页面”。
6. 点击“查看结果”。
7. 在结果页点击某个账户的“打开主页”。
8. 在该账户主页中再次点击插件图标。
9. 点击“读取当前主页最近发帖时间”。
10. 回到结果页查看状态。
11. 手动决定是否在 X 页面取关。
12. 对不想重复处理的账户，可以标记“已处理”或加入“白名单”。

## 使用建议

1. 每次只扫描当前已经展示出来的账户。
2. 不要长时间连续高频操作。
3. 遇到验证或访问限制时立即停止。
4. 取关前请手动打开主页确认。
5. 白名单用于保留重要账户。
6. 已处理用于避免重复检查。
7. 建议将 X 页面语言切换为英文后读取最近发帖时间。

## 设置

点击 Popup 中的“打开设置”可以调整：

1. 未活跃判断天数：30 / 60 / 90 / 180。
2. 是否隐藏白名单账户。
3. 是否显示无法判断账户。
4. 默认排序。
5. X 页面语言提示。

设置只保存在本地，不会上传。

## 数据导出

结果页支持导出 CSV 和 JSON。

CSV 字段：

```csv
username,displayName,profileUrl,lastPostAt,inactiveDays,status,processed,whitelisted,collectedAt,lastCheckedAt,manualNote
```

JSON 包含本地账户数组、设置和导出时间，不包含 Cookie、Token、密码、私信或浏览器会话信息。

## 清空数据

Popup 中点击“清空本地数据”可以清空本地保存的账户列表。此操作只影响插件本地数据，不会影响你的 X 账号。

## 常见问题

### 插件会自动取关吗？

不会。插件只做辅助整理，所有取关都必须由你在 X 页面手动决定和操作。

### 插件会读取我的 Cookie、Token、密码或私信吗？

不会。Manifest 没有申请 `cookies`、`webRequest`、`debugger`、`history`、`downloads` 等高风险权限。

### 为什么扫描不到账户？

可能原因：

1. 当前页面不是 Following 页面。
2. Following 页面还没有加载账户。
3. 你还没有登录 X。
4. X 出现验证或访问限制。
5. X 页面 DOM 结构变化。

### 为什么主页最近发帖时间无法判断？

可能原因：

1. 账户没有公开帖子。
2. 账户受保护。
3. 账户被暂停或不存在。
4. 页面语言不是英文，时间格式暂未覆盖。
5. X 页面结构变化。

### 可以批量自动检查所有账户吗？

MVP 不包含该功能。为了降低账号风险，当前版本只支持用户手动打开主页后读取当前可见内容。

## 账号风险提醒

本项目不能承诺完全没有账号风险。为了降低风险，请保持低频、手动、只读、可见操作。出现验证、访问限制或页面异常时，请立即停止使用插件。

## 开发检查

本项目不需要构建步骤。可以运行：

```sh
npm run check
```

该命令会检查必需文件、Manifest 禁止权限和 `src` 下 JavaScript 语法。
